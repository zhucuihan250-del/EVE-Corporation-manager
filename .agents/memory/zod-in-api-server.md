---
name: Zod imports in api-server
description: How to use Zod validation in api-server routes without importing zod directly
---

# Rule
Never import `zod` or `zod/v4` directly in `artifacts/api-server/src/routes/*.ts`. esbuild cannot resolve these as direct imports because `zod` is not in api-server's dependencies.

# How to apply
Use the generated Zod schemas from `@workspace/api-zod`:
```typescript
import { CreateFleetBody, GetFleetParams } from "@workspace/api-zod";
```

These are generated by Orval from the OpenAPI spec (`pnpm --filter @workspace/api-spec run codegen`). Always add new request body schemas to the OpenAPI spec and run codegen before writing new routes.

**Why:** esbuild bundles the api-server and cannot resolve `zod` because it's not a direct dependency. Other workspace packages that use zod (api-zod, db) bundle it themselves.
